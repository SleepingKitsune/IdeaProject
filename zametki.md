API к сущностям
1. PICTURE
## 1. Получить все картинки
**Метод:** `GET`
**URL:** `/picture/`
**Ответ:**
- `200` — массив картинок.
- `404` — картинки не найдены.

## 2. Получить картинку по ID

**Метод:** `GET`
**URL:** `/picture/:id`
**Параметры:**
- `id` — идентификатор картинки (число).
**Ответ:**
- `200` — данные картинки.
- `400` — некорректный ID.
- `404` — картинка не найдена.

## 3. Создать новую картинку
**Метод:** `POST`
**URL:** `/picture/`
**Заголовки:** `Content-Type: multipart/form-data`
**Тело запроса:**
- `picture_name` — название картинки (строка).
- `idIdea` — ID идеи, к которой привязывается картинка.
- `pictureIdea` — файл картинки (обязателен).
**Ответ:**
- `201` — картинка успешно сохранена.
- `400` — не указаны название, ID идеи или не загружен файл.
- `404` — идея с указанным ID не найдена.

## 4. Удалить картинку
**Метод:** `DELETE`
**URL:** `/picture/:id`
**Параметры:**
- `id` — идентификатор картинки (число).
**Ответ:**
- `200` — картинка успешно удалена.
- `404` — картинка не найдена.

## Примечания
- Все ошибки сопровождаются JSON с описанием проблемы.
- Для загрузки файлов используется `multer` (multipart/form-data).

> **Важно:** при удалении картинки связь с идеей автоматически разрывается, идея не удаляется.

- Для создания категории требуется только поле `category_name`.

LIKE

## 1. Поставить лайк идее

**Метод:** `POST`
**URL:** `/like`
**Тело запроса (JSON):**
```json
{
  "idUser": 1,
  "idIdea": 42
}
```
**Ответ:**
*   `201` — Лайк успешно поставлен.
*   `400` — Не указаны ID пользователя или идеи.
*   `404` — Пользователь или идея не найдены.
*   `409` — Лайк уже поставлен.

## 2. Удалить лайк

**Метод:** `DELETE`
**URL:** `/like/:id`
**Параметры:**
*   `id` — идентификатор лайка (число).
**Ответ:**
*   `200` — Лайк успешно удалён.
*   `400` — Некорректный ID.
*   `404` — Лайк с таким ID не найден.

## 3. Получить количество лайков у идеи

**Метод:** `GET`
**URL:** `/like/count/idea/:id`
**Параметры:**
*   `id` — идентификатор идеи (число).
**Ответ:**
*   `200` — Успешно. Возвращает объект с количеством.
    ```json
    { "count": 15 }
    ```
*   `400` — Некорректный ID идеи.

## Примечания
*   Для работы с лайками требуется авторизация пользователя (токен). В запросах на постановку лайка ID пользователя будет определяться автоматически из токена, а не передаваться в теле запроса. Текущий код — базовая реализация.

Краткое руководство по эндпоинтам. Все методы, кроме `/registration` и `/auth`, требуют авторизации (наличия сессии).
## 1. Регистрация и Авторизация

### Регистрация нового пользователя
*   **Метод:** `POST`
*   **URL:** `user/registration`
*   **Тело запроса:** `{ "nickname", "email", "password" }`
*   **Ответ:**
    *   `201` — Регистрация успешна.
    *   `400` — Некорректные данные.
    *   `409` — Пользователь с таким email уже существует.

### Авторизация (Вход в систему)
*   **Метод:** `POST`
*   **URL:** `user/auth`
*   **Тело запроса:** `{ "email", "password" }`
*   **Ответ:**
    *   `200` — Авторизован. В ответе устанавливается `Cookie` `Session`.
    *   `401` — Неверный email или пароль.

---

## 2. Получение данных

### Получить данные одного пользователя по ID
*   **Метод:** `GET`
*   **URL:** `user/:id`
*   **Ответ:**
    *   `200` — Успешно. Возвращает объект с полями `id`, `nickname`, `email`, `aboutMe`.
    *   `400` — Некорректный ID.
    *   `400` / `404` — Пользователь не найден.

### Получить список всех пользователей
*   **Метод:** `GET`
*   **URL:** `user/`
*   **Ответ:**
    *   `200` — Успешно. Возвращает массив объектов.
    *   `404` — Пользователи не найдены.

---

## 3. Обновление данных (Требуется авторизация)

Все следующие запросы требуют наличия активной сессии (куки `Session`). Если пользователь не авторизован, вернётся ошибка `401`.

### Обновить профиль (Никнейм и описание)
*   **Метод:** `PUT`
*   **URL:** `user/updateUser`
*   **Тело запроса:** `{ "nickname", "aboutMe" }`
*   **Ответ:**
    *   `200` — Профиль успешно обновлен.

### Обновить email
*   **Метод:** `PUT`
*   **URL:** `user/updateEmail`
*   **Тело запроса:** `{ "email" }`
*   **Ответ:**
    *   `200` — Email успешно обновлен.

### Обновить пароль
> ***Важно:*** *В теле запроса должны быть оба пароля.*
*   **Метод:** `PUT`
*   **URL:** `user/updatePassword`
*   **Тело запроса:** `{ "oldPassword", "newPassword" }`
*   **Ответ:**
    *   `200` — Пароль успешно обновлен.
    *   `401` — Неверный текущий пароль.

### Обновить аватар
> ***Важно:*** *Загрузка файла.*
*   **Метод:** `PUT`
*   **URL:** `user/updateAvatar`
*   **Заголовки:** `Content-Type: multipart/form-data`
*   **Тело запроса:** Файл с ключом `avatar`.
*   **Ответ:**
    *   `200` — Аватар успешно обновлен.

---

## 4. Удаление данных

### Удалить пользователя по ID
> ***Важно:*** *Этот метод удаляет пользователя по ID из URL, а не текущего авторизованного пользователя.*
*   **Метод:** `DELETE`
*   **URL:** `user/:id`
*   **Ответ:**
    *   `200` — Пользователь успешно удален.
    *   `401` / `404` — Пользователь не найден или нет прав на удаление.

# API для работы с идеями (Idea)

Краткое руководство по эндпоинтам. Все методы, кроме просмотра (`GET`), требуют авторизации (наличия активной сессии).

## 1. Получение данных

### Получить одну идею по ID
*   **Метод:** `GET`
*   **URL:** `/idea/:id`
*   **Ответ:**
    *   `200` — Успешно. Возвращает объект идеи.
    *   `404` — Идея не найдена.

### Получить список всех идей
*   **Метод:** `GET`
*   **URL:** `/idea`
*   **Ответ:**
    *   `200` — Успешно. Возвращает массив всех идей.
    *   `404` — Идеи не найдены.

## 2. Создание и Изменение (Требуется авторизация)

Все следующие запросы требуют наличия активной сессии (куки `Session`).

### Создать новую идею
*   **Метод:** `POST`
*   **URL:** `/idea`
*   **Тело запроса (JSON):**
    ```json
    {
      "category": 1,
      "name_idea": "Название идеи",
      "desc": "Описание идеи",
      "need_part": true,
      "status": "searching"
    }
    ```
*   **Ответ:**
    *   `201` — Идея успешно создана.
    *   `401` — Вы не авторизованы.
    *   `400` — Некорректные данные (нет названия или описания).

### Обновить существующую идею
*   **Метод:** `PUT`
*   **URL:** `/idea/:id`
*   **Тело запроса (JSON):**
    *   Можно отправлять только те поля, которые нужно изменить.
    ```json
    {
      "name_idea": "Новое название"
    }
    ```
*   **Ответ:**
    *   `200` — Идея успешно обновлена.
    *   `401` — Вы не авторизованы.
    *   `403` — У вас нет прав на редактирование этой идеи.
    *   `404` — Идея не найдена.

### Удалить идею
*   **Метод:** `DELETE`
*   **URL:** `/idea/:id`
*   **Ответ:**
    *   `200` — Идея успешно удалена.
    *   `401` — Вы не авторизованы.
    *   `403` — У вас нет прав на удаление этой идеи.
    *   `404` — Идея не найдена.

## 3. Управление главной картинкой

### Установить главную картинку для идеи
Этот метод позволяет выбрать, какая из загруженных картинок будет отображаться как основная.

*   **Метод:** `PUT`
*   **URL:** `/idea/:id/setMainPicture`
*   **Тело запроса (JSON):**
    ```json
    {
      "pictureId": "имя_файла.jpg" // или ID картинки из БД
    }
    ```
*   **Ответ:**
    *   `200` — Главная картинка успешно установлена.
    *   `401` — Вы не авторизованы.
    *   `403` — У вас нет прав на изменение этой идеи.
    *   `404` — Идея не найдена.
